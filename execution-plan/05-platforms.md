# 05 — Platform Pages (§1–4)

Four flagship products. Each gets a parent page + sub-module pages. All inherit the
**multi-brand template** with platform-specific color theming.

## Shared platform template

Every platform parent page follows this section order:

1. **Hero** — brand-tinted background, H1 tagline (from blueprint), subhead, dual CTAs, product mockup
2. **What it does** — 4–5 bullets summary (lifted from blueprint "WHAT X DOES")
3. **Module / capability grid** — 2×3 or 3×2 cards with Lucide icon + title + description
4. **Positioning vs alternative** — short comparison call-out where relevant
5. **Target customer band** — chips listing customer types (D2C, EdTech, etc.)
6. **Sub-page anchor strip** — pills linking to deep-dive sub-pages
7. **Featured integrations / partners** — logo cloud filtered by platform relevance
8. **Pricing teaser** — single most-relevant tier card + link to /pricing
9. **Bottom CTA band** — brand-colored, two CTAs
